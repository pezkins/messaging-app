## How to Use This Documentation

### For AI Agents
All agents should reference these documents before making changes:
1. Check `project-overview.md` for architectural context
2. Review `api-contracts.md` for API specifications
3. Follow `coding-standards.md` for consistency
4. Understand `release-process.md` for deployment
5. Track progress in `feature-roadmap.md`

### For Human Developers
1. Read `project-overview.md` when onboarding
2. Reference `api-contracts.md` for integration work
3. Follow `coding-standards.md` for code reviews
4. Use `release-process.md` for deployments

## Updating Documentation

- **Architect/Reviewer Agent:** Has write access to all documentation
- **Other Agents:** Should suggest documentation updates in their responses
- **Human Developers:** Can update any documentation via PR

## Documentation Standards

1. Use clear, concise language
2. Include code examples where helpful
3. Keep documentation up-to-date with code changes
4. Use consistent formatting and structure
5. Date significant changes
